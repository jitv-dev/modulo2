Amenazas comunes

Técnica de ingeniería social que intenta engañar a las víctimas para que revelen información sensible como credenciales de acceso o datos financieros, normalmente a través de correos electrónicos, mensajes o sitios web falsos que imitan entidades legítimas.

  • Suplantación de identidad (bancos, servicios populares, compañías)
  • Urgencia o amenazas en el mensaje
  • Enlaces a sitios web falsos
  • Archivos adjuntos maliciosos
  • Errores gramaticales o de diseño

  • Verificar siempre la dirección del remitente
  • No hacer clic en enlaces sospechosos
  • Comprobar la URL de los sitios web
  • Usar autenticación de dos factores

Malware que cifra los archivos o sistemas de la víctima y exige un pago (generalmente en criptomonedas) a cambio de liberarlos.

  • Bloqueo total o parcial de sistemas
  • Exigencia de rescate con plazo límite
  • Amenazas de destrucción de datos
  • Propagación a través de redes

  • Mantener copias de seguridad actualizadas
  • Actualizar sistemas y software
  • No pagar el rescate (no garantiza recuperación)
  • Usar soluciones antiransomware

Software malicioso diseñado para dañar sistemas, robar información o tomar control de dispositivos sin el consentimiento del usuario.

  • Virus: Se adjunta a archivos limpios y se replica
  • Troyanos: Se disfrazan de software legítimo
  • Spyware: Recopila información sin consentimiento
  • Adware: Muestra publicidad no deseada

  • Instalar y mantener antivirus actualizado
  • Descargar software solo de fuentes oficiales
  • Evitar archivos adjuntos sospechosos
  • Usar cuentas de usuario con privilegios limitados

Ataque que sobrecarga un sistema, servidor o red con tráfico masivo desde múltiples fuentes, haciendo que el servicio sea inaccesible para usuarios legítimos.

  • Implementar soluciones anti-DDoS
  • Configurar balanceadores de carga
  • Usar servicios de mitigación en la nube
  • Monitorizar el tráfico de red

  • Instalar y mantener antivirus actualizado
  • Descargar software solo de fuentes oficiales
  • Evitar archivos adjuntos sospechosos
  • Usar cuentas de usuario con privilegios limitados

Ejemplos casos reales

SolarWinds (2020)

Ataque de cadena de suministro que comprometió software empresarial para espiar a miles de organizaciones, incluyendo agencias gubernamentales de EE.UU.

Colonial Pipeline (2021)

Ataque de ransomware que paralizó el mayor oleoducto de combustible de EE.UU., causando escasez y pagando $4.4 millones en rescate.

Kaseya (2021)

Ataque ransomware que afectó a más de 1,500 empresas a través de un proveedor de software de gestión.

Log4j (2021)

Vulnerabilidad crítica en una biblioteca de Java ampliamente utilizada que permitió ejecución remota de código en millones de sistemas.

Facebook (2021)

Exposición de datos personales de más de 530 millones de usuarios debido a vulnerabilidades en sistemas de la plataforma.
